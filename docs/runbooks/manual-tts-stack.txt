Ebook Audiobook Pipeline Manual Runtime Notes
Date: 2026-06-20

Purpose
-------
This runbook records the manual prerequisites for the v1 Python-first audiobook pipeline.

V1 does not use LM Studio and does not require a manually started Qwen FastAPI server. The pipeline calls Anthropic directly for annotation and loads Qwen3-TTS through a local Python wrapper for TTS.

Required Services And Credentials
---------------------------------
1. Anthropic
   - Create or use an Anthropic API key.
   - Set ANTHROPIC_API_KEY in the shell before running the pipeline.
   - Default model is Sonnet. The exact pinned model ID lives in pipeline config and can be changed without editing code.

2. Qwen3-TTS local runtime
   - Install the Python dependencies required by the Qwen TTS wrapper from flybirdxx/ComfyUI-Qwen-TTS.
   - Make sure the Qwen3-TTS model files are available locally.
   - Recommended local model path layout:

     models/
       qwen-tts/
         Qwen3-TTS-12Hz-1.7B-Base/
         Qwen3-TTS-12Hz-1.7B-VoiceDesign/
         Qwen3-TTS-Tokenizer-12Hz/

     The adapter also accepts the ComfyUI layout with an extra Qwen/ directory under qwen-tts.

   - Use bf16 precision and auto attention by default when the hardware supports it.

3. Book files
   - Put a whole-book text file at:

     books/<book_slug>/source/book.txt

   - Or put pre-split chapters at:

     books/<book_slug>/chapters/chapter_001.txt
     books/<book_slug>/chapters/chapter_002.txt

Runtime Order
-------------
1. Prepare the environment.
   - Activate the Python environment for the pipeline.
   - Confirm ANTHROPIC_API_KEY is set.
   - Confirm local Qwen model files exist.

2. Run ingestion and segmentation.
   - If source/book.txt exists, run deterministic chapter splitting.
   - Run deterministic sentence segmentation for each chapter.
   - Confirm files exist under:

     books/<book_slug>/sentence_segments/

3. Run quote attribution.
   - The pipeline extracts quoted dialogue spans directly from the raw chapter text.
   - Anthropic maps each quote ID to either a global registry role or a chapter-local temporary speaker.
   - Narrator text is everything outside quote spans and is assembled deterministically during script generation.
   - Chapter-local temporary speakers are not written to the global registry.

4. Run voice preparation.
   - For every role in the chapter, the pipeline checks registry.json.
   - If voice_config_path points to a readable .qvp file, reuse it.
   - If no .qvp exists, generate one once from the saved voice profile and voice identity seed.
   - Save .qvp files under:

     books/<book_slug>/voices/

5. Run chapter TTS.
   - The pipeline builds role-tagged Qwen sections with at most 8 roles and a default 1100-character section limit.
   - Override section size with EBOOK_TTS_MAX_TTS_WINDOW_CHARS only when changing annotation/TTS section shape. This is not the main VRAM throttle.
   - Saved TTS sections are generation checkpoints. Within each section, Qwen still receives a multi-item voice_clone_prompt call for mixed roles unless an explicit generation guard splits that section.
   - EBOOK_TTS_QWEN_MAX_GENERATION_BLOCKS_PER_CALL defaults to 0, which means no static block-count split. Set it only as a temporary safety cap while investigating a risky section.
   - Qwen peak VRAM is driven by generation-call shape: generated audio/token length, number of role-run prompt items, max_new_tokens, attention backend, and hidden-state/KV-cache retention. Character count is only a rough proxy.
   - Use EBOOK_TTS_QWEN_PERF_LOG to write one JSONL row per Qwen generation call with section indices, section shape, text lengths, sample counts, elapsed time, and CUDA allocated/reserved/peak memory when torch.cuda is available.
   - Sentence timelines include the actual generated voice_config_path used for each audio span.
   - Sentences are atomic. A sentence that does not fit in the current window moves to the next window.
   - A single sentence that is too large for a window causes a clear failure.
   - When a split occurs inside a contiguous speaker block, the next section repeats the speaker role tag so every section is self-contained.

6. Inspect outputs.
   - Annotation:

     books/<book_slug>/annotations/chapter_001.annotation.json

   - Audio:

     books/<book_slug>/audio/chapter_001.wav

   - Sentence timing:

     books/<book_slug>/audio/chapter_001.timeline.json

Consistency Rule
----------------
For consistent character voices across chapters, reuse saved .qvp files. Do not regenerate a voice from only the text voice description unless the user intentionally wants a new voice.

Character Profile Rule
----------------------
The registry treats age-stage versions of the same underlying person as separate character profiles. If the text reveals the same person as a child, teen, adult, or elder, create separate profile IDs such as:

  callie_teen
  callie_adult
  trevor_child
  andrew_adult

Use the same person_id to link those profiles when the text supports that they are the same underlying person. Do not force one voice profile across life stages; age stage, experience, and personality changes should produce separate voice files.

New-character LLM output should be compact. It should provide identity fields only:

  age_stage, gender, personality, race_or_ethnicity, accent, occupation, person_id, profile_id, aliases

The registry generates the Qwen voice prompt from age stage, gender, personality, occupation, and text-grounded race/accent. Relationship facts and plot facts should not be stored in the registry or qwen_instruct.

If abuse, grooming, exploitation, coercion, or child sexual abuse appears in the source text, never describe it as romance or consensual adult intimacy. Keep that context out of voice/profile fields unless a future safety-specific UI field is added.

Troubleshooting
---------------
- Missing ANTHROPIC_API_KEY:
  Set the environment variable before running annotation.

- Low-confidence chapter split:
  Provide pre-split chapter files or configure a custom chapter heading regex.

- Invalid LLM JSON:
  The pipeline should retry once with a repair prompt. If it still fails, inspect the saved failed response.

- Missing or unreadable .qvp:
  Regenerate the voice only if the existing file is absent or intentionally replaced.

- Similar voices for different characters:
  Check registry.json for voice_identity.differentiators. Distinct characters should have stable differentiators such as timbre, cadence, resonance, or energy.

- Highlight timing looks wrong:
  Check chapter_001.timeline.json. Timing should be computed from generated waveform sample counts and sample rate before the final WAV is written.

V1 CLI Commands
---------------
Install local package for development:

  python -m pip install -e ".[dev]"

Install Qwen runtime dependencies:

  python -m pip install -e ".[qwen]"

For NVIDIA CUDA on Windows, install the matching PyTorch CUDA wheels after the qwen extra. On the tested RTX 5080 Laptop GPU setup:

  python -m pip install --force-reinstall torch==2.8.0+cu129 torchaudio==2.8.0+cu129 --index-url https://download.pytorch.org/whl/cu129

The tested Qwen runtime versions are:

  qwen3-tts-comfyui 1.0.7
  torch 2.8.0+cu129
  torchaudio 2.8.0+cu129
  transformers 4.57.3
  numpy 1.26.4
  hf_xet installed for Hugging Face Xet-backed model files

Optional environment overrides:

  $env:EBOOK_TTS_QWEN_MODEL_ROOT="models/qwen-tts"
  $env:EBOOK_TTS_QWEN_MODEL="1.7B"
  $env:EBOOK_TTS_QWEN_DEVICE="cuda"
  $env:EBOOK_TTS_QWEN_PRECISION="bf16"
  $env:EBOOK_TTS_QWEN_ATTENTION="auto"
  $env:EBOOK_TTS_QWEN_MAX_NEW_TOKENS="2048"
  $env:EBOOK_TTS_QWEN_MAX_GENERATION_BLOCK_CHARS="0"
  $env:EBOOK_TTS_QWEN_MAX_GENERATION_BLOCKS_PER_CALL="0"
  $env:EBOOK_TTS_QWEN_ADAPTIVE_TARGET_GB="13"
  $env:EBOOK_TTS_QWEN_PERF_LOG="logs/qwen_generation_perf.jsonl"

For speed tuning on a 16 GB GPU, start with EBOOK_TTS_QWEN_PERF_LOG enabled and set EBOOK_TTS_QWEN_ADAPTIVE_TARGET_GB to the desired working ceiling, such as 13. The adapter records max_memory_reserved/max_memory_allocated per generation call and, when an adaptive target is set, adjusts the next call's block limit from the measured peak. The first metrics to compare are tts_section_indices, generation_block_count, text_char_sum, sample_counts/audio_seconds, elapsed_seconds, adaptive_next_blocks_per_call, and CUDA peak memory. Leave EBOOK_TTS_QWEN_MAX_GENERATION_BLOCKS_PER_CALL and EBOOK_TTS_QWEN_MAX_GENERATION_BLOCK_CHARS at 0 unless the logs identify a specific risky section that needs a temporary guard. Lower EBOOK_TTS_QWEN_MAX_NEW_TOKENS only if logs show runaway or near-limit generations, because setting it too low can truncate valid long speech.

Install FlashAttention 2 when supported by the GPU/runtime. With EBOOK_TTS_QWEN_ATTENTION=auto, the adapter uses flash_attention_2 when flash_attn is importable and bf16 precision is selected; otherwise it falls back to sdpa.

Download local Qwen model folders:

  python scripts/download_qwen_models.py --model-root models/qwen-tts --base-model 1.7B

If Hugging Face Xet requests repeatedly reset, retry with regular HTTP:

  python scripts/download_qwen_models.py --model-root models/qwen-tts --base-model 1.7B --disable-xet

Run a minimal real Qwen smoke test without Anthropic:

  python scripts/smoke_qwen_tts.py --book-root books/qwen_smoke --regenerate-voice

Run all tests:

  python -m pytest -v

Run a chapter with fake TTS for pipeline validation:

  ebook-tts run-chapter --book-root books/demo --book-title "Demo" --book-slug demo --chapter chapter_001 --fake-tts

Run a chapter with real Qwen TTS after local Qwen wiring is complete:

  ebook-tts run-chapter --book-root books/demo --book-title "Demo" --book-slug demo --chapter chapter_001

Run individual pipeline steps for UI-style workflows:

  ebook-tts segment-chapter --book-root books/demo --chapter chapter_001
  ebook-tts annotate-chapter --book-root books/demo --book-title "Demo" --book-slug demo --chapter chapter_001
  ebook-tts build-tts-script --book-root books/demo --chapter chapter_001
  ebook-tts prepare-voices --book-root books/demo --chapter chapter_001
  ebook-tts synthesize-chapter --book-root books/demo --chapter chapter_001

If the user edits only character voice profiles and wants to regenerate audio, keep the saved TTS text script and run only:

  ebook-tts synthesize-chapter --book-root books/demo --chapter chapter_001

If the user edits annotation, sentence-role assignments, or sentence text, rebuild the TTS script before audio:

  ebook-tts build-tts-script --book-root books/demo --chapter chapter_001
  ebook-tts synthesize-chapter --book-root books/demo --chapter chapter_001

To force synthesis to rebuild the TTS script from the current annotation in one command:

  ebook-tts synthesize-chapter --book-root books/demo --chapter chapter_001 --rebuild-tts-script

To force voice `.qvp` regeneration from the current saved voice profiles:

  ebook-tts synthesize-chapter --book-root books/demo --chapter chapter_001 --regenerate-voices
